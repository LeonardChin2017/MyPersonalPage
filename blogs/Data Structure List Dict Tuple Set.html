Python: List vs Tuple vs Set vs Dict

Published on Oct 26, 2021

Introduction

List, Tuple, Set, and Dictionary are fundamental data types in Python, each serving specific purposes based on the requirements of a given task. Selecting the appropriate data type is crucial as it can significantly impact execution time and memory efficiency. This blog will explore these sequence and mapping data types and their respective use cases.

List

A list is a collection of items arranged in a specific order. You can access items in a list using numbers, like 'python_list[3]'. The time it takes for various operations with a list depends on the list's size, which we'll call 'n'. Lists function like dynamic arrays in C++, where each item has a position in the array, and the size can change. Lists are handy for storing items, even if there are duplicates, and they preserve the order of the items. Also, you can change the contents of a list after creating it (mutable).

Example:

            
                python_list = ['one', 2, 3.4, 'hello']
                print(python_list)
                print(python_list[3])
                python_list[1] = 2
                print(python_list)

Output:

            
                ['one', 2, 3.4, 'hello']
                hello
                [2, 2, 3.4, 'hello']

Tuple

Tuples are similar to lists because they can store different data types. However, there's a key difference: tuples are immutable, which means you can't change their elements once you've assigned them. In contrast, lists are dynamic and can be modified.In simpler terms, think of a tuple as an "unchangeable" list. You can't add, remove, or replace elements in a tuple once it's created. Everything in a tuple stays the same. However, you can create a new tuple by combining two or more tuples.In Python, we typically use tuples to store data that doesn't need to change. Tuples don't have methods like "append" or "extend" for adding elements, and there are no "remove" or "pop" methods for removing elements. You can still find elements in a tuple without altering the tuple itself. You can use the "in" operator to check if an element exists in a tuple.Another advantage of tuples is that they are faster than lists. If you have a set of constant values that you'll only iterate through and never modify, using a tuple is a good choice.

Example:

            
                python_tuple = (1, 'raju', 28, 'abc')
                print(python_tuple)
                print(python_tuple[1])

Output:

            
                (1, 'raju', 28, 'abc')
                raju

Set

In Python, a Set is a collection that doesn't allow duplicates. It's excellent for quickly checking if something is in it (compared to lists; it's much faster), but it doesn't maintain any order. Sets are also useful for mathematical operations and removing duplicates.

Example:

            
                python_set_1 = {1, 2, 3}
                print(python_set)
                python_set_2 = set(['foo', 'bar', 'baz', 'foo', 'qux'])
                print(python_set_2)

Output:

            
                {1, 2, 3}
                {'qux', 'foo', 'bar', 'baz'}

Dictionary

In Python, a dictionary functions like a hash table, associating keys with values. It serves as a key-value storage type, similar to a real dictionary where values are connected to keys. Dictionaries are changeable, enabling you to add, modify, or remove key-value pairs. Unlike lists and sets, which solely contain values, dictionaries establish relationships between keys and values. If you need an efficient way to look up values using specific keys, dictionaries are the right choice.

Example:

            
                python_dict = {"Nishant": 1, "Akash": 2, "Ravi": 3, "Hari": 4}
                print(python_dict)
                print(python_dict["Akash"])

Output:

            
                {'Nishant': 1, 'Akash': 2, 'Ravi': 3, 'Hari': 4}
                2
