Data Structure: Stack vs Queue

Published on Oct 22, 2021

Introduction

Queue and stack are actually concepts that describe how you manage your data in a data structure. Under the hood, they can be implemented with either an array or a linked list. A queue follows the 'First In, First Out' (FIFO) principle, while a stack follows the 'Last In, First Out' (LIFO) principle.

Stack:

Fundamentally, whenever you need to reverse the order of elements and retrieve them in constant time, you can use a stack. A stack follows the LIFO principle, which is just a way of organizing data.

Applications:

  1. Achieving the 'undo' operation in notepads.
  2. The browser's back button uses a stack.

Example:

In C++, you can use the stack library: https://cplusplus.com/reference/stack/stack/

In Python, you can use list as stack.

Queue:

Queues are implemented using a FIFO principle.

Applications:

  1. In real life, call center phone systems use queues to hold people calling them in order until a service representative is free.
  2. CPU scheduling and disk scheduling. When multiple processes require the CPU at the same time, various CPU scheduling algorithms are used, which are implemented using queue data structures.
  3. Print spooling.
  4. Breadth-First Search in a graph.
  5. Handling interrupts in real-time systems. The interrupts are handled in the same order as they arrive, following a 'First Come, First Served' approach.

Example:

In C++, you can use the queue library: https://en.cppreference.com/w/cpp/container/queue< /p>

In Python, you can use collection.deque as queue: https://docs.python.org/3/library/collections.html#deque-objects

Summary

You can use a queue when you want to retrieve items in the order you added them, and a stack when you prefer to retrieve items in the reverse order of how you added them.
